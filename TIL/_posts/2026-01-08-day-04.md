## Iterating over Python dictionaries properly
Iterating over a dictionary directly returns its keys by default.
Using .keys() in a for loop is unnecessary when only keys are needed.
To access both keys and values efficiently, use .items().
.items() improves readability and avoids extra dictionary lookups.

### Best practices:
Use for key in dict: instead of for key in dict.keys()
Use for key, value in dict.items() when both are needed
```python
for key in dict:
    print(key)
for key, value in dict.items():
    print(key, value)
```


## Iterating over multiple lists in Python
Using range(len()) to iterate over lists works but is not Pythonic.
enumerate() is useful when you need both the index and the value.
When iterating over two lists in parallel, zip() is the cleanest and most readable solution.
zip() avoids index-based access and reduces the risk of errors.

### Best practice:
Use enumerate() for index + value
Use zip() for multiple sequences
```python
for i, country in enumerate(countries):
    population = populations[i]
    print(country,population)

for country, population in zip(countries, populations):
    print(country, population)
```
