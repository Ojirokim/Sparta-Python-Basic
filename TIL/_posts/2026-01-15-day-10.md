## MySQL REGEXP (TIL)

### What REGEXP does
- `REGEXP` is used to **match strings against a pattern**
- It returns `TRUE` or `FALSE` for each row
- Syntax:
```sql
column REGEXP 'pattern'
```

### Regex building blocks (most common)
Anchors
^ â†’ start of string
$ â†’ end of string

Example:
```sql
^abc     -- starts with "abc"
abc$     -- ends with "abc"
```

### Character classes
[A-Z] â†’ uppercase letters
[a-z] â†’ lowercase letters
[A-Za-z] â†’ any letter
[0-9] â†’ digits
[A-Za-z0-9_] â†’ alphanumeric + underscore

Example:
```sql
^[A-Za-z]   -- string must start with a letter
```

### Quantifiers
* â†’ zero or more
+ â†’ one or more
? â†’ zero or one

Example:
```sql
[A-Za-z0-9]*   -- any number of allowed characters
```

### Special characters
. â†’ any character
\. â†’ literal dot
- â†’ literal dash (inside [])

Example: email validation pattern
```sql
^[A-Za-z][A-Za-z0-9_.-]*@leetcode\.com$
```
Meaning:
^ â†’ start
[A-Za-z] â†’ must start with a letter
[A-Za-z0-9_.-]* â†’ allowed characters
@leetcode\.com â†’ exact domain
$ â†’ end

### Case sensitivity in MySQL REGEXP
MySQL REGEXP is case-insensitive by default
This behavior comes from the column collation
Most collations end with _ci (ci = case-insensitive)

### How to make REGEXP case-sensitive
REGEXP_LIKE
```sql
WHERE REGEXP_LIKE(mail, 'leetcode\\.com', 'c')
```
'c' â†’ case-sensitive
'i' â†’ case-insensitive
