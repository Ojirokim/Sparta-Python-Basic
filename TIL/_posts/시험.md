### DW의 모델링 기법

- 스타 스키마
  조인 스키마라고도 하며, 가장 단순하다.
  하나의 사실 테이블을 중심으로 여러개의 차원 테이블 존재
  전통적인 관계형 DBMS를 통해서 다차원 데이터베이스 기능 구현 가능
  사실 테이블 = 제 3정규형
  차원 테이블 = 제 2정규형(비정규화됨)
  장점: 복잡도 낮아서 이해 쉬움, 쿼리작성 용이, 조인 테이블 수가 적음
  단점: 차원 테이블이 비정규화되어 데이터 중복으로 인한 데이터 적재 시간 오래 걸림.

- 스노플레이크 스키마
  스타 스키마에서 차원테이블을 제 3정규화한 형태
  장점: 데이터의 중복이 제거되어 적재 시간이 줄어듬
  단점: 스키마 구조의 복잡성이 증가되어 조인 테이블 수가 증가하고 쿼리 작성 난이도가 높음

### 가상머신의 메모리 할당

1) Memory Ballooning
   VMkernal은 over-using하는 가상머신의 메모리 영역을 빈값으로 강제로 채워서 가상머신 운영체제가 자체적으로 swapping 하도록 한다. Swap된 메모리를 하이퍼바이저가 타 가상머신에 할당

2) Transparent Page Sharing
   하나의 물리적 머신에 동일한 내용을 담은 페이지는 물리적인 메로리 영역 하나에만 존재시키고 가상머신들이 공유

3) Memory Overcommitment
   2GB물리적메모리에 5개의 512MB 수행.
   => 권장하지 않는 방법이다.

   예측ture정답/ 예측true

### 인공신경망 활성화 함수

1. 계단 함수: 0 이상은 1, 이외는 0
2. 시그모이드 함수: S자형으로 모든 값이 0과 1사이.
3. ReLU 함수: 0 이상은 입력값 그대로, 이외는 0
4. Leaky ReLU: 0 이상은 입력값 그대로, 이외는 엄청작은 수로(0.01) -> 뉴런이 죽는 현상 해결위해 나옴
5. 하이퍼볼릭 탄젠트 함수: 시그모이드를 변형하여 s의 중심점을 0으로 옮김. -1에서 1사이값.
6. Softmax 함수: 표준화지수 함수라고도 불리며, 출력값이 여러 개로 주어지고 목표치가 다범주인 경우, 사후확률 제공

### 암묵지 vs 형식지
1) 공통화: 암묵지 -> 암묵지 ★★★
예) A가 B에게 구두로 노하우를 인계

2) 표출화: 암묵지 -> 형식지
예) A가 노하우를 글로 작성해서 전달하는 것

3) 내면화: 형식지 -> 암묵지
예) B가 글로 작성된 노하우를 체화하는 것

4) 연결화: 형식지 -> 형식지 ★★★
예) A와 B의 문서가 합쳐져 더 좋은 문서로 되었다.
